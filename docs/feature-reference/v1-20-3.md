# Features from Minecraft 1.20.3

Java **1.20.3** added **decorated pot** storage behavior, the **`/tick`** command (freeze, step,
query, rate control, sprint), and an **experimental Crafter** preview. **Redstone Backport** on
**1.20.1** implements **`/tick`** and the associated tick-rate managers; it does **not** implement
decorated pots or the Crafter from this release (the **Crafter** is backported from **1.21** on a
separate page).

:::{important}
Decorated pots from 1.20.3 are **not** included in this mod as of the current documentation build.
See the {ref}`version matrix <version-matrix>`.
:::

## `/tick` command (vanilla reference)

- **Permission:** ops level **3** (`Commands.LEVEL_ADMINS`), not 4.
- **Subcommands:** `query`, `rate`, `freeze` / `unfreeze`, `step` (+ `stop`), `sprint` (+ `stop`).

Minecraft **1.20.3** introduced **`TickRateManager`** and **`ServerTickRateManager`** (not
“ServerTickManager”) and threaded them through the server loop and world ticking.

### Vanilla client behavior

Newer vanilla sends **`ClientboundTickingStatePacket`** and **`ClientboundTickingStepPacket`**. Stock
**1.20.1** clients do not understand those IDs, so this mod uses **custom S2C channels** instead
(same state, loader-native networking).

## What this mod implements

### Server

- **Managers:** `TickRateManager`, `ServerTickRateManager` in `common` — sprint, step, freeze, rate.
- **Command:** `TickCommand` plus `commands.tick.*` lang entries.
- **Duck API:** `TickRateManagerAccess` on `MinecraftServer` (mixin): tick manager + average tick time
  derived from `long[] tickTimes` (1.20.1 has no vanilla `getAverageTickTimeNanos()`).
- **`MinecraftServerMixin`:** Manager init; `tick()` at `tickServer` HEAD; dynamic tick delay
  (`@ModifyConstant` on `50L`); sprint via `BooleanSupplier` redirect and `endTickWork`.
  **`onTickRateChanged()`** is intentionally empty (1.20.1 autosave cadence is not tied to tick rate
  like newer vanilla).
- **`ServerLevelMixin`:** `@Redirect` gates world border, weather, `tickTime`, raids, dragon, block
  events, scheduled block/fluid ticks (via `isDebug`), when `!runsNormally()`.
- **`ServerChunkCacheMixin`:** Skips `tickChunks` when frozen.
- **`LevelEntityFreezeMixin`:** `Level.guardEntityTick` — skips non-player entities per
  `isEntityFrozen` on **server and client**.
- **`PlayerListMixin`:** Syncs tick state to joining players (`placeNewPlayer`).
- **Loaders:** Fabric/Quilt `CommandRegistrationCallback`; Forge `RegisterCommandsEvent`.
- **Mixins:** `redstone_backport.mixins.json` + `redstone_backport.client.mixins.json`; Forge registers
  both in the jar manifest.

**Entity freezing note:** vanilla 1.20.3+ uses `countPlayerPassengers() <= 0` for “is this entity
frozen”; the 1.20.1 backport uses **`getIndirectPassengers()`** (no player in the indirect passenger
set).

### Client sync

- **S2C:** `TickingStatePayload` (rate + frozen) and `TickingStepPayload` (`frozenTicksToRun`).
- **Handlers:** Fabric/Quilt: `ClientPlayNetworking`; Forge: `SimpleChannel`.
- **Client manager:** `ClientTickRateAccess` on `ClientLevel` (`ClientLevelMixin`): mirrors server
  state.
- **Gates:** Client `tickTime()` gated when frozen; entity freeze on client.
- **Join:** State sent on join so late joiners match `/tick` state.

:::{warning}
**Mixin compatibility:** server-side **`@Redirect`** targets inside `ServerLevel.tick()` and
**`@ModifyConstant`** on the server sleep interval compete with **at most one** other mod per
bytecode site. If another mod redirects the same `INVOKE` or constant, mixins may conflict. Prefer
testing modpacks that also tweak global ticking. Client-side work is mostly **`@Inject`** and is
usually easier to combine.
:::

### Optional / deferred parity (not required for normal use)

:::{note}
The following match **newer vanilla** only if you need the last few percent of parity. Gameplay for
`/tick` is complete.
:::

- **`ServerPlayer` / collision:** possible vanilla tweaks when pushing frozen entities — not mixined
  here unless issues appear.
- **`ServerLevel.tick` audit:** remaining body vs 1.20.4 if something still advances while “frozen.”
- **`onTickRateChanged` / autosave:** newer vanilla can tie autosave spacing to tick rate; 1.20.1 stub
  stays empty.
- **Integrated server / pause:** 1.20.1 already pauses singleplayer on Esc; no extra hook required.

## Testing

- **Dedicated server + mcrcon:** strong fit for server `/tick` and gate mixins.
- **Unit tests:** high value for manager state (`freeze`, `step`, sprint, `runsNormally()`).
- **Packet codecs:** medium — round-trip `TickingStatePayload` / `TickingStepPayload`.
- **Full client CI:** low — use a short **manual** checklist (freeze, reconnect, multiplayer) unless
  you add heavyweight display CI.

## Misc

**Not a reference:** “OverClocked: Clockworks”-style MCreator commands are **not** Mojang-parity
`/tick`; this mod follows vanilla structure.
